Get started with View Exports SVG by installing it from the Visual Studio Code Marketplace. The installation process takes just a few seconds.

## System requirements

Before installing View Exports SVG, ensure your system meets the following requirements:

<CardGroup cols={2}>
  <Card title="VS Code version" icon="code">
    Version 1.109.0 or higher
  </Card>
  
  <Card title="Supported platforms" icon="laptop">
    Windows, macOS, and Linux
  </Card>
</CardGroup>

<Note>
View Exports SVG requires VS Code version **1.109.0 or higher**. You can check your current version by going to **Help > About** in VS Code.
</Note>

## Install from VS Code Marketplace

The easiest way to install View Exports SVG is directly from within Visual Studio Code.

<Steps>
  <Step title="Open the Extensions view">
    Press `Ctrl+Shift+X` (Windows/Linux) or `Cmd+Shift+X` (macOS) to open the Extensions view in VS Code.
    
    Alternatively, click the Extensions icon in the Activity Bar on the left side of the window.
  </Step>
  
  <Step title="Search for View Exports SVG">
    In the Extensions view search box, type `View Exports SVG`.
    
    Look for the extension published by **JairTorres1003** with the distinctive icon.
  </Step>
  
  <Step title="Click Install">
    Click the **Install** button next to the View Exports SVG extension.
    
    The installation typically completes in a few seconds.
  </Step>
  
  <Step title="Verify installation">
    Once installed, you should see the extension listed in your installed extensions.
    
    You can verify the installation by opening the Command Palette (`Ctrl+Shift+P` or `Cmd+Shift+P`) and searching for "View Exports SVG" commands.
  </Step>
</Steps>

## Install from the marketplace website

You can also install the extension from the Visual Studio Code Marketplace website.

<Steps>
  <Step title="Visit the marketplace page">
    Open your web browser and navigate to the [View Exports SVG marketplace page](https://marketplace.visualstudio.com/items?itemName=JairTorres1003.jt-view-exports-svg).
  </Step>
  
  <Step title="Click Install">
    Click the **Install** button on the marketplace page.
    
    This will prompt your browser to open Visual Studio Code.
  </Step>
  
  <Step title="Confirm installation in VS Code">
    VS Code will open with the extension page displayed.
    
    Click **Install** to complete the installation.
  </Step>
</Steps>

## Install from VSIX file

For offline installation or to install a specific version, you can use a VSIX file.

<Steps>
  <Step title="Download the VSIX file">
    Download the `.vsix` file from the [GitHub releases page](https://github.com/JairTorres1003/JT-View-Exports-SVG/releases) or the marketplace.
  </Step>
  
  <Step title="Open the Extensions view">
    Press `Ctrl+Shift+X` (Windows/Linux) or `Cmd+Shift+X` (macOS) to open the Extensions view.
  </Step>
  
  <Step title="Install from VSIX">
    Click the three-dot menu ("...") at the top of the Extensions view.
    
    Select **Install from VSIX...**
  </Step>
  
  <Step title="Select the VSIX file">
    Navigate to the downloaded `.vsix` file and select it.
    
    VS Code will install the extension and prompt you to reload the window.
  </Step>
</Steps>

## Verify installation

After installation, you can verify that View Exports SVG is working correctly:

1. Open the Command Palette (`Ctrl+Shift+P` or `Cmd+Shift+P`)
2. Type "View Exports SVG" - you should see commands like:
   - **View Exports SVG: Start Scanning**
   - **View Exports SVG: Reload Theme**
   - **View Exports SVG: Clear Cache**

<Tip>
If you don't see these commands, try reloading VS Code by running **Developer: Reload Window** from the Command Palette.
</Tip>

## Update the extension

View Exports SVG receives regular updates with new features and bug fixes.

<Steps>
  <Step title="Check for updates">
    VS Code automatically checks for extension updates. When an update is available, you'll see an update button in the Extensions view.
  </Step>
  
  <Step title="Install updates">
    Click the **Update** button next to View Exports SVG to install the latest version.
    
    Alternatively, enable automatic updates in VS Code settings.
  </Step>
</Steps>

## Troubleshooting

<Accordion title="Extension not showing up after installation">
  Try reloading VS Code by running **Developer: Reload Window** from the Command Palette (`Ctrl+Shift+P` or `Cmd+Shift+P`).
</Accordion>

<Accordion title="Minimum VS Code version error">
  View Exports SVG requires VS Code 1.109.0 or higher. Update your VS Code installation by going to **Help > Check for Updates**.
</Accordion>

<Accordion title="Installation fails">
  If installation fails, try:
  - Restarting VS Code
  - Clearing the extension cache
  - Installing from a VSIX file instead
  - Checking your internet connection
</Accordion>
